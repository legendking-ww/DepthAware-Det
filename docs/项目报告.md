# DepthAware-Det 项目说明报告

**项目名称**：DepthAware-Det — 基于单目视觉的检测与深度融合 3D 感知系统  
**文档类型**：项目说明 / 技术报告  
**适用场景**：课程设计、毕业设计说明、作品集技术文档  

---

## 一、项目概述

### 1.1 项目做什么

DepthAware-Det 是一套面向**单目摄像头**的轻量级 **3D 场景感知**软件系统。系统在普通 RGB 图像或视频上同时完成：

1. **2D 目标检测**（车辆、行人等）；  
2. **稠密/半稠密深度估计**（每个像素的相对远近）；  
3. **检测目标级别的深度融合**（为每个检测框赋予深度值）；  
4. **几何尺度校准**（将模型输出的相对深度转换为具有物理意义的**米制距离**）；  
5. **鸟瞰图（BEV）可视化**（将目标投影到地面平面，便于理解空间布局）。

用户可通过**命令行实时程序**或 **Web 网页界面**上传图片、视频或打开摄像头，即时查看带检测框、深度标注与 BEV 小窗的结果，无需激光雷达或多目相机。

### 1.2 解决什么问题

| 问题 | 本项目的做法 |
|------|----------------|
| 仅有 2D 检测框，不知道目标多远 | 结合单目深度网络，在框内提取深度 |
| 深度网络输出为相对值，无绝对米制 | 利用相机焦距 + 物体典型高度估计尺度因子 |
| 算法演示门槛高，依赖命令行 | 提供 Gradio Web 界面与一键启动脚本 |
| 实时性不足 | 支持 ONNX / TensorRT 加速，全链路 GPU 推理 |

### 1.3 项目定位

- **类型**：工程实践型视觉感知系统（非端到端自动驾驶全栈）。  
- **目标用户**：计算机视觉学习者、课程/毕设演示、算法原型验证。  
- **不涵盖**：完整 SLAM、轨迹规划、控制决策；米制精度依赖先验与标定，非计量级 LiDAR 替代。

## 二、技术方案总览

### 2.1 技术栈

| 层次 | 技术 | 作用 |
|------|------|------|
| 目标检测 | **YOLOv8**（Ultralytics） | 实时 2D 检测，输出类别与边框 |
| 深度估计 | **Depth Anything V2**（ViT-S） | 单目相对深度图，518×518 输入 |
| 深度学习框架 | **PyTorch 2.x** | 训练权重加载、预处理与 TRT 缓冲 |
| 推理加速 | **ONNX Runtime GPU** | 跨平台 GPU 推理 |
| 推理加速 | **TensorRT 8.6** | FP16 引擎，最低延迟 |
| 图像处理 | **OpenCV** | 读写、resize、可视化 |
| Web 界面 | **Gradio** | 图片/视频/摄像头交互 |
| 语言与环境 | **Python 3.10**、**Anaconda**、**CUDA** | 开发与运行环境 |

### 2.2 模型与数据流

```
RGB 图像
    ├─→ YOLOv8 ─────────────→ 检测框列表 [x1,y1,x2,y2,conf,cls]
    │
    └─→ Depth Anything V2 ──→ 深度图 H×W（相对深度）
                │
                ▼
        框中心区域深度融合（中值 / IQR）
                │
                ▼
        多目标尺度校准（焦距 + 类别先验高度）
                │
                ▼
        米制深度标注 + BEV 鸟瞰图绘制
```

### 2.3 三种推理后端

| 后端 | 模型形式 | 特点 |
|------|----------|------|
| PyTorch | `.pt` + `.pth` | 部署简单，便于调试 |
| ONNX Runtime | `.onnx` | 无需 TensorRT，GPU 加速明显 |
| TensorRT | `.engine`（FP16） | 延迟最低，适合实时 Demo |

检测与深度模块可统一选择同一后端；Web 界面与命令行程序共用同一套接口（`create_detector` / `create_depth_estimator`）。

## 三、系统功能说明

### 3.1 核心功能

| 功能编号 | 功能名称 | 说明 |
|----------|----------|------|
| F1 | 多类别目标检测 | 支持车、人、骑行者等 COCO 类别，可调置信度与输入分辨率 |
| F2 | 单目深度估计 | 整图深度图，采用官方 ImageNet 归一化与多尺度 resize 策略 |
| F3 | 检测框深度融合 | 在框中心 50% 区域内取鲁棒统计量，抑制边缘与背景干扰 |
| F4 | 米制尺度校准 | 基于针孔模型 \( d = f \cdot H / h_{px} \) 与多目标中值估计尺度 \(s\) |
| F5 | BEV 鸟瞰可视化 | 将目标底边中心按深度与内参投影至地面，绘制俯视图 |
| F6 | 实时视频/摄像头 | 支持隔帧深度更新、HUD、深度热力图画中画 |
| F7 | KITTI 评估（可选） | 与稀疏 LiDAR 投影深度对比 abs_rel、RMSE |

### 3.2 应用与交互功能

| 功能 | 入口 | 说明 |
|------|------|------|
| Web 图片推理 | `app.py` → 图片 Tab | 上传街景图，输出检测 + 深度条 + BEV |
| Web 视频推理 | `app.py` → 视频 Tab | 上传视频，生成标注视频，可限最大帧数 |
| Web 实时摄像头 | `app.py` → 摄像头 Tab | 浏览器授权后流式推理 |
| 命令行实时 Demo | `scripts/run_realtime.py` | 本地摄像头/视频文件，`d`/`q` 快捷键 |
| 单图快速测试 | `scripts/test_single_image.py` | 验证管线是否正常 |
| 环境与模型自检 | `scripts/check_env.py` | 检查 CUDA、TRT、ONNX 及模型文件 |
| 一键启动 | `启动网页.bat` / `launch_web.ps1` | 自动配置路径并启动 Web |

### 3.3 工程辅助功能

- **ONNX 导出**：`scripts/export_onnx.py` 统一导出 YOLO 与 Depth 模型。  
- **TensorRT 构建**：`scripts/build_trt.py`，Windows 下自动处理中文路径（临时目录中转）。  
- **模型缺失提示**：`src/model_check.py` 在启动前检查并给出中文修复建议。  
- **Git 友好**：大权重与 engine 不纳入版本库，克隆后按文档生成。

## 四、关键技术说明

### 4.1 检测模块

- 采用 **YOLOv8s**，在精度与速度之间平衡。  
- TensorRT 模式下加载 `yolov8s_fp16.engine`，由 Ultralytics 调用 TensorRT 运行时。  

### 4.2 深度模块

- 采用 **Depth Anything V2 ViT-S**（`vits`），输入分辨率 **518×518**。  
- 预处理与官方 `infer_image` 一致，避免简单除以 255 导致深度异常。  
- TensorRT 实现类 `DepthEstimatorTRT` 使用 PyTorch CUDA 缓冲区 + `execute_async_v3` 执行引擎。

### 4.3 深度融合与校准

- **融合**：在检测框中心区域计算深度中值，可选 IQR 剔除离群点。  
- **校准**：对已知典型高度的类别（车、人、车等），由像素高度与焦距估计几何深度，再与模型相对深度求比例，取多目标中值作为全局尺度因子。  
- **输出**：每个检测框显示如 `car 12.3m` 的标签。

### 4.4 BEV 模块

- 取检测框**底边中心**作为接地点近似。  
- 结合焦距 \(f_x, f_y\)、主点 \(c_x, c_y\) 与米制深度，将目标投影至地面坐标系并绘制鸟瞰图。  
- 在实时界面中以**画中画**形式叠加，不遮挡主画面。

## 五、系统架构

### 5.1 软件分层

```
┌─────────────────────────────────────────┐
│  交互层：Gradio Web / OpenCV 窗口        │
├─────────────────────────────────────────┤
│  应用层：app.py、run_realtime.py         │
├─────────────────────────────────────────┤
│  管线层：DepthAwarePipeline              │
├─────────────────────────────────────────┤
│  算法层：detector / depth / fusion /     │
│          calibration / bev               │
├─────────────────────────────────────────┤
│  推理层：PyTorch | ONNX Runtime | TRT    │
└─────────────────────────────────────────┘
```

### 5.2 主要目录职责

| 目录/文件 | 职责 |
|-----------|------|
| `src/pipeline.py` | 串联检测、深度、融合、校准、BEV |
| `src/detector.py` | YOLO 封装与后端路径解析 |
| `src/depth_estimator.py` | 三种深度后端 |
| `src/fusion.py` | 框内深度提取 |
| `src/calibration.py` | 尺度因子与 KITTI 标定解析 |
| `src/bev.py` | 鸟瞰图生成 |
| `src/utils.py` | 绘制、HUD、画中画合成 |
| `app.py` | Web 服务与模型缓存 |

## 六、开发与运行环境

| 项目 | 推荐配置 |
|------|----------|
| 操作系统 | Windows 10/11（脚本针对 PowerShell 优化，Linux 可跑核心代码） |
| GPU | NVIDIA GPU，示例：RTX 4060 Laptop（8GB） |
| CUDA | 12.1（与 PyTorch wheel 匹配） |
| TensorRT | 8.6.1（可选，用于最快推理） |
| Python | 3.10（conda 环境 `depthaware`） |

在本机实测（TensorRT FP16）：深度网络约 **4.3 ms/帧**（518×518）；端到端 FPS 还受 YOLO、绘制与显示影响，可通过 `--depth-every 2` 提升。

## 七、使用流程摘要

1. 安装 Conda 环境与 `requirements.txt`。  
2. 下载深度权重：`python scripts/download_weights.py`。  
3. 导出 ONNX：`python scripts/export_onnx.py --all`。  
4. （可选）构建 TensorRT：`python scripts/build_trt.py --all`。  
5. 自检：`python scripts/check_env.py`。  
6. 启动 Web：双击 `启动网页.bat` 或 `python app.py`。  

详细命令见项目根目录 [`README.md`](../README.md)。

## 八、项目成果与创新点（可作答辩/简历表述）

1. **检测与深度统一管线**：将通用 2D 检测与 SOTA 单目深度模型工程化串联，形成可复现的 3D 感知 Demo。  
2. **相对深度到米制的轻量校准**：不依赖 LiDAR 在线标定，利用多目标先验高度与焦距完成尺度恢复。  
3. **多后端推理架构**：同一业务代码切换 PyTorch / ONNX / TensorRT，便于开发、部署与性能对比。  
4. **完整产品化交付**：Web 界面、环境自检、一键启动、模型检查与 Git 大文件分离，具备可展示、可迁移的工程形态。

## 九、局限与后续工作

| 局限 | 说明 |
|------|------|
| 米制精度 | 依赖类别先验高度与近似焦距，复杂场景误差较大 |
| 地面假设 | BEV 假设平坦地面，坡道/高架场景偏差明显 |
| 评估数据 | KITTI 完整评测需用户自行准备数据集 |
| 测试覆盖 | 尚未建立自动化单元测试与 CI |

**可扩展方向**：多帧时序平滑、可学习尺度网络、接入 ROS2、部署至 Jetson、与跟踪模块结合等。

## 十、总结

DepthAware-Det 是一个**以单目 RGB 为输入、融合目标检测与深度估计的 3D 感知演示系统**。项目在算法上结合 YOLOv8 与 Depth Anything V2，在工程上实现多后端加速、尺度校准、BEV 可视化与 Web 交互，适用于教学演示、算法验证与个人作品集展示。
